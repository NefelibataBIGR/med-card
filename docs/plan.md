# 医学教材抽卡复习网页方案

## Summary
- 构建一个本地运行的 Web 应用，使用 `Python 虚拟环境 + Python 后端 + 前端单页应用 + SQLite`。
- 用户一次导入一本教材 PDF，系统先抽取文本，再调用 `DeepSeek API` 识别“概念 + 精简释义 + 来源章节”，生成可复习卡片。
- 复习时按“未抽取优先、模糊池高优先、单轮不重复”规则抽卡；卡片可标记为“熟悉”或“模糊”，分别进入熟悉卡池和模糊卡池。
- 不做单独导入校对页，但抽出的卡片支持 `修改 / 删除 / 忽略 / 保存`，保证错误抽取可以在使用中修正。

## Key Changes
- 技术栈默认定为：
  - 后端：`FastAPI`
  - 前端：`React + Vite`
  - 数据库：`SQLite`
  - PDF 处理：`pypdf` 为主，必要时预留 OCR 扩展位
  - LLM 接口：`DeepSeek`，通过兼容 OpenAI 风格的客户端封装
- 运行方式固定为虚拟环境：
  - 项目根目录创建 `.venv`
  - 所有 Python 命令通过 `.venv` 内解释器执行
  - 后端依赖写入 `requirements.txt`
- 核心流程：
  1. 上传单本 PDF。
  2. 后端抽取全文并按章节/段落切块。
  3. 将文本分批发送给 DeepSeek，要求输出结构化 JSON：`concept_name`、`summary`、`chapter`、`source_excerpt`。
  4. 后端做去重、清洗、长度限制和落库。
  5. 前端进入抽卡页，点击抽取一张未重复卡片。
  6. 用户对当前卡片执行 `熟悉 / 模糊 / 忽略 / 编辑保存 / 删除`。
  7. 熟悉卡进入熟悉池，模糊卡进入模糊池；模糊卡在后续抽取中提高权重。
- 数据模型最少包含：
  - `textbook`：教材导入记录、文件名、导入时间、处理状态
  - `card`：概念名、精简释义、来源章节、原文片段、状态、忽略标记、更新时间
  - `review_log`：每次抽取与选择结果，用于避免单轮重复和后续扩展
  - `session_state`：当前复习轮次中已抽出的卡片集合
- 卡片状态规则：
  - `new`：新卡，未进入任一卡池
  - `familiar`：熟悉池
  - `uncertain`：模糊池
  - `ignored`：忽略，不参与抽卡
  - 删除为硬删除或软删除二选一，默认采用软删除更安全
- 抽卡算法固定为：
  - 单次复习轮次内不重复
  - 优先从 `uncertain` 中抽取
  - 若模糊池为空，再从 `new` 中抽取
  - 熟悉池默认不参与常规抽卡，只在查阅页展示
  - 当 `uncertain + new` 都抽完时，提示本轮完成并允许重开新一轮
- 页面结构固定为：
  - `导入页`：上传 PDF、查看处理进度、查看导入结果摘要
  - `抽卡页`：卡面展示、抽取按钮、熟悉/模糊/忽略/编辑/删除操作
  - `卡池页`：熟悉卡池、模糊卡池分栏浏览与搜索
  - `卡片详情弹层`：编辑概念名、释义、章节并保存
- API 设计最少包含：
  - `POST /api/textbooks/import`
  - `GET /api/textbooks`
  - `GET /api/cards/draw`
  - `PATCH /api/cards/{id}`
  - `POST /api/cards/{id}/mark-familiar`
  - `POST /api/cards/{id}/mark-uncertain`
  - `POST /api/cards/{id}/ignore`
  - `DELETE /api/cards/{id}`
  - `GET /api/pools/familiar`
  - `GET /api/pools/uncertain`
- LLM 抽取约束固定为：
  - 输出严格 JSON
  - 每个概念必须包含名称和不超过设定长度的精简释义
  - 同一章节内相近概念做后端去重
  - 保留 `source_excerpt` 便于后续人工修正
  - 抽取失败的文本块进入重试队列并记录错误

## Test Plan
- PDF 导入：
  - 能成功上传单本 PDF 并生成教材记录
  - 文本块抽取异常时能返回明确错误
  - LLM 某一批失败时不影响已成功批次落库
- 卡片生成：
  - 同名或高度相似概念不会重复入库
  - 卡片字段缺失或过长时会被清洗或丢弃
  - 忽略卡不会出现在抽卡结果中
- 抽卡逻辑：
  - 单轮内不会重复抽到同一张卡
  - 模糊池存在时优先抽到模糊卡
  - 模糊卡标为熟悉后，应从模糊池移入熟悉池
  - `new + uncertain` 抽空后会正确提示轮次结束
- 编辑与卡池：
  - 抽出的卡可以修改并持久化保存
  - 删除后卡片不再参与抽卡和卡池展示
  - 熟悉池、模糊池都支持列表查看与基础搜索
- 运行与配置：
  - 在 `.venv` 中完成依赖安装和后端启动
  - `.env` 中缺少 `DeepSeek API Key` 时，后端能给出明确配置错误

## Assumptions
- 第一版只支持 `有文本层` 的 PDF；OCR 仅预留扩展接口，不在首版强制实现。
- DeepSeek 作为默认抽取模型，但代码层保留兼容 OpenAI/Qwen 风格接口的提供者抽象。
- 前端不做登录、多用户、云同步；全部数据保存在本机 `SQLite`。
- 熟悉池默认仅供查阅，不参与常规抽卡；模糊池参与优先复习。
- 建议方案文档文件名定为 `docs/plan.md`，后续如果切到执行模式，再按该内容落盘并开始搭建项目。
